## 2025-06-13 - Optimize Supabase Query Performance with asyncio.gather and Batching
**Learning:** Sequential synchronous database calls in FastAPI create significant N+1 query latency because the backend currently uses a synchronous Supabase client. In nested relations like fetching a full resume where there are multiple parent and child tables (e.g. experiences -> bullets), the delay stacks up quickly.
**Action:** Use `asyncio.to_thread` combined with `asyncio.gather` for parallel fetching of top-level relationships. For child records, pluck the parent IDs and perform batch fetching using an `.in_()` clause instead of looping query calls, then stitch the parent and child records back together in memory using `collections.defaultdict`. Update callers to support the asynchronous operation.
